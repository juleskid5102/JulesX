# /qa - Quality Assurance & Security

## Prerequisites
- Đã hoàn thành Development (Phase 4)
- Code build thành công (`npm run build` không lỗi)

## Bước 1: Code Quality Review

### 1.1 TypeScript kiểm tra
// turbo
```bash
npx tsc --noEmit
```
- Không được có `any` type (trừ trường hợp đặc biệt)
- Mọi component phải có Props interface
- Mọi function phải có return type

### 1.2 Lint check
// turbo
```bash
npm run lint
```

### 1.3 Code Review bằng skill
Sử dụng skill `code-reviewer` và `find-bugs`:
- Review tất cả files trong `src/`
- Tìm code smells, anti-patterns
- Kiểm tra error handling

**Checklist code quality:**
- [ ] Không có `console.log` dư thừa
- [ ] Không có TODO/FIXME chưa xử lý
- [ ] Không có hardcoded values
- [ ] Error handling cho mọi async operation
- [ ] Loading states cho data fetching
- [ ] Empty states khi không có data

## Bước 2: Security Audit

### 2.1 Firebase Security Rules
Kiểm tra `firebase/firestore.rules`:
- [ ] Không có `allow read, write: if true` (mở toang)
- [ ] Mọi collection đều có rules
- [ ] User chỉ đọc/ghi data của mình
- [ ] Admin routes có kiểm tra role
- [ ] Validate data structure trước khi write

### 2.2 Frontend Security
Sử dụng skill `frontend-security-coder`:
- [ ] Không lưu secrets ở frontend
- [ ] XSS protection (không dùng `dangerouslySetInnerHTML`)
- [ ] CSRF protection
- [ ] Input validation (Zod schemas)
- [ ] Sanitize user input trước khi hiển thị

### 2.3 Auth Security
- [ ] Password requirements đủ mạnh
- [ ] Session timeout hợp lý
- [ ] Protected routes kiểm tra auth state
- [ ] Không expose user data qua URL

### 2.4 Environment Variables
- [ ] `.env.local` trong `.gitignore`
- [ ] Không hardcode API keys
- [ ] Production env vars set trên hosting

## Bước 3: Performance Testing

### 3.1 Build & Bundle Analysis
// turbo
```bash
npm run build
```

Kiểm tra:
- [ ] Build thành công
- [ ] Bundle size < 500KB (gzipped)
- [ ] Không import thư viện nặng không cần thiết

### 3.2 Lighthouse Audit
Sử dụng skill `playwright-skill` hoặc browser:
- [ ] Performance score ≥ 90
- [ ] Accessibility score ≥ 90
- [ ] Best Practices score ≥ 90
- [ ] SEO score ≥ 90

### 3.3 Core Web Vitals
- [ ] LCP (Largest Contentful Paint) < 2.5s
- [ ] FID (First Input Delay) < 100ms
- [ ] CLS (Cumulative Layout Shift) < 0.1

### 3.4 Image Optimization
- [ ] Images sử dụng format tối ưu (WebP/AVIF)
- [ ] Images có lazy loading
- [ ] Images có width/height attributes (tránh CLS)

## Bước 4: Visual Validation

### 4.1 So sánh với Stitch designs
Sử dụng skill `ui-visual-validator`:
- Mở mỗi page trên browser
- So sánh với screenshots trong `designs/screens/`
- Check: colors, spacing, typography, layout match

### 4.2 Responsive Testing
Test trên các breakpoints:
- [ ] Mobile (375px)
- [ ] Tablet (768px)
- [ ] Desktop (1440px)
- [ ] Large screen (1920px)

### 4.3 Cross-browser
- [ ] Chrome
- [ ] Firefox
- [ ] Safari (nếu có thể)

## Bước 5: Functional Testing

### 5.1 Happy Path
Test mỗi user flow quan trọng:
- [ ] Home page load
- [ ] Navigation giữa các pages
- [ ] Form submission
- [ ] Login/Register (nếu có)
- [ ] CRUD operations (nếu có)

### 5.2 Edge Cases
- [ ] Empty states (không có data)
- [ ] Error states (network error, invalid input)
- [ ] Loading states
- [ ] Long text overflow
- [ ] Special characters trong input

## Bước 7: Fix Issues

Sửa tất cả issues severity High và Medium trước khi deploy.
Issues Low có thể track để sửa sau release.

> **Khi QA pass → chuyển sang Phase 6: `/deploy`**
